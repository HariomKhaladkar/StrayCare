# StrayCare Android App

Built with **Capacitor 6** — wraps the existing React web app as a native Android app.

## Architecture

```
android-app/
├── android/              ← Native Android Studio project
├── capacitor.config.json ← App config + Camera plugin settings
├── package.json          ← Capacitor + @capacitor/camera dependencies
└── node_modules/
```

The Android app is a **Capacitor wrapper** around the React frontend. All web features are automatically available on Android after a `cap sync`:

| Feature | Android Status |
|---------|---------------|
| Recovery Stories Feed | ✅ Works (same React app) |
| User Profile Page | ✅ Works |
| NGO Public Profile | ✅ Works |
| In-App Notifications 🔔 | ✅ Works (30s polling) |
| Food Marketplace 🛒 | ✅ Works |
| **Native Camera** 📷 | ✅ NEW — uses `@capacitor/camera` |

---

## 📷 Native Camera Feature

On Android, the **Report Case** photo upload uses the native camera instead of a file picker:

- **Tap the camera button** → system prompt appears: "Take a Photo" or "Choose from Gallery"
- Photo is processed by the on-device **AI animal recognition** (MobileNet) immediately after capture
- If the app is on a browser/web, it falls back to the standard styled file upload

**Permissions required** (already declared in `AndroidManifest.xml`):
- `CAMERA`
- `READ_MEDIA_IMAGES` (Android 13+)
- `READ_EXTERNAL_STORAGE` (Android ≤ 12)
- `ACCESS_FINE_LOCATION` (for GPS on case report)

## How to Build the APK

### Prerequisites
- [Android Studio](https://developer.android.com/studio) installed (Hedgehog or later)
- [JDK 17+](https://www.oracle.com/java/downloads/)
- Node.js 18+

### Step 1 — Install dependencies (first time only)

```bash
# Frontend dependencies
cd frontend
npm install

# Android wrapper dependencies
cd ../android-app
npm install
```

### Step 2 — Set the backend IP in the frontend

Edit `frontend/src/api.js` and set your machine's local IP:

```js
const API_BASE_URL = 'http://YOUR_IP:8000';
```

Also update `android-app/capacitor.config.json`:
```json
"url": "http://YOUR_IP:3000"
```

### Step 3 — Build and sync (every time you update the web app)

```bash
# 1. Build the React frontend
cd frontend
npm run build

# 2. Sync the new build into the Android project
cd ../android-app
npx cap sync
```

### Step 4 — Open in Android Studio

```bash
cd android-app
npx cap open android
```

Android Studio opens → **Build → Build APK(s)** to generate the APK.

### Step 5 — Run on a device or emulator

In Android Studio: Click the **▶ Run** button.

Or via ADB (with device connected + USB debugging enabled):

```bash
# Build debug APK
cd android-app/android
./gradlew assembleDebug

# Install on connected device
adb install app/build/outputs/apk/debug/app-debug.apk
```

## Granting Camera Permissions on Device

On first launch, Android will ask for **Camera** and **Location** permissions.
- Tap **Allow** for Camera (required for photo upload in Report Case)
- Tap **Allow only while using the app** for Location

If permissions were denied, go to:
> **Settings → Apps → StrayCare → Permissions → Camera / Location → Allow**

## Notes

- Files inside `android/app/src/main/assets/public/` are auto-generated by `cap sync` — never edit them manually
- UPI deep-links (`upi://pay?...`) work natively on Android — all payment options function inside the app
- The live development server URL in `capacitor.config.json` is only used during development; the production APK uses the bundled build from `frontend/build/`
